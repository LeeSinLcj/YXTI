# ACGTI 第一轮校准计划

## 目标

第一轮只做小范围、可验证、可回滚的权重修正，目标是优先抬高 `S/N` 和 `J/P`：

- `S/N`：从当前高置信约 `55.3%` 提升到 `58%–61%`
- `J/P`：从当前高置信约 `58.1%` 提升到 `61%–64%`
- 不改题面
- 不改 `dimension`
- 不改 `sign`
- 不自动套用全部逻辑回归权重

## 输入

- 数据报告：`analysis/reports/feedback_analysis_summary_2026-04-20.md`
- 题目权重映射：`analysis/reports/top_weight_questions_mapped.csv`
- 人工复核表：`analysis/reports/question_review_round1.csv`

## 第一轮权重范围

只处理 10 道候选题，其中实际覆盖 8 道维度算分权重，保留 2 道稳定锚点。

实现位置：

```text
src/data/questionDimensionWeights.json
```

该文件只影响 MBTI 四维算分强弱，不改变题面、原始题目归属字段和原型/角色权重。

默认情况下，每题仍按：

```text
question.dimension + question.sign
```

计分。只有出现在 `questionDimensionWeights.json` 的题会使用覆盖权重。

## 角色层范围

第一轮角色层只做保守处理，不大洗牌，不删除角色。

低一致率候选：

```text
MARN, ERUC, TMRS, NENE, TAFI, CIRN, MYRI, SERN, MEGR, NEUR, TNAO, KALS, RIKI
```

建议动作：

1. 只降低命中优先级或候补权重。
2. 不修改角色文案、不批量改 MBTI。
3. 优先复核“样本量 >= 20 且一致率低”的角色。
4. 如果已有 `matchWeight` 机制，第一轮只对低一致率角色轻微降权，不让概率文件被脚本全量重写。

## 版本标记

下一版必须 bump app version，例如：

```text
0.3.1-sn-jp-calibration1
```

对比指标：

- 高置信完整四字母一致率
- `S/N` 一致率
- `J/P` 一致率
- 高置信反馈数量
- 低一致率角色是否改善

## 不做事项

- 不全自动套用 `weights_*.csv`
- 不改题面
- 不改 `dimension/sign`
- 不全量重排角色池
- 不让脚本直接覆盖线上概率文件
- 不把分析文件、题目修正、角色降权混成一个不可拆提交
